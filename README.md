### Βάση Δεδομένων

Το αρχείο **`sql/university_system.sql`** περιλαμβάνει το **export** της βάσης δεδομένων.
Οι βασικοί πίνακες είναι:

* users
* roles
* courses
* assignments
* submissions
* enrollments

Η βάση δεδομένων χρησιμοποιεί:

* foreign keys
* unique constraints
* `ON DELETE CASCADE`

για τη διασφάλιση της ακεραιότητας των δεδομένων.

* Hashing κωδικών χρηστών
* PDO prepared statements
* PHP sessions

### Περιγραφή βασικών αρχείων

**index.html:**
Η κεντρική σελίδα της εφαρμογής. Περιλαμβάνει συνδέσμους προς τις σελίδες login και register, χάρτη με χρήση Leaflet (εμπλουτισμένο μέσω `js/main.js`) και τρεις επιπλέον συνδέσμους που οδηγούν στις σελίδες:
`lab.html`, `library.html`, `sports.html`

**lab.html:**
Περιλαμβάνει δύο κουμπιά (εργαστήρια υπολογιστών και ιατρικά εργαστήρια). Με την επιλογή τους, ο χρήστης μπορεί να δει πληροφορίες και λεπτομέρειες για τα αντίστοιχα εργαστήρια.

**library.html:**
Παρουσιάζει εικόνες και πληροφορίες σχετικά με τις εγκαταστάσεις και τις παροχές της πανεπιστημιακής βιβλιοθήκης προς τους φοιτητές.

**sports.html:**
Παρουσιάζει εικόνες και πληροφορίες σχετικά με τις αθλητικές εγκαταστάσεις που προσφέρει το πανεπιστήμιο.

## Αρχεία PHP

**config.php:**
Περιλαμβάνει τη σύνδεση με τη βάση δεδομένων και τις απαραίτητες συναρτήσεις που χρησιμοποιούνται από όλα τα PHP αρχεία της εφαρμογής.

**register.php:**
Επιτρέπει στον χρήστη να εγγραφεί εισάγοντας username, email, password, επιλογή ρόλου (Φοιτητής ή Καθηγητής) και κωδικό εγγραφής (`STUD2025` ή `PROF2025`). Σε περίπτωση λανθασμένων ή ελλιπών στοιχείων εμφανίζεται κατάλληλο μήνυμα. Μετά από επιτυχή εγγραφή, ο χρήστης μεταφέρεται στη σελίδα login.php.

**login.php:**
Περιλαμβάνει τον έλεγχο σύνδεσης χρήστη. Σε επιτυχή σύνδεση, ο χρήστης μεταφέρεται στο `dashboard.php`, διαφορετικά εμφανίζεται μήνυμα σφάλματος.

**logout.php:**
Πραγματοποιεί αποσύνδεση του χρήστη και καταστροφή του session.

**dashboard.php:**
**Κοινό** dashboard για Φοιτητές και Καθηγητές. Κάθε ρόλος έχει πρόσβαση μόνο στις αντίστοιχες λειτουργίες του. Σε απόπειρα πρόσβασης σε μη επιτρεπόμενη σελίδα εμφανίζεται μήνυμα **“Forbidden Action”**.
Παράδειγμα: Αν ένας Καθηγητής προσπαθήσει να αποκτήσει πρόσβαση στη διεύθυνση
`/prototype/php/student/view_courses.php`
εμφανίζεται το παραπάνω μήνυμα.

**header.php / footer.php:**
Περιλαμβάνουν τον κώδικα για το header και το footer της εφαρμογής, με responsive σχεδιασμό.

## Ρόλοι Χρηστών & Λειτουργίες

### Φοιτητής

* **Προβολή διαθέσιμων μαθημάτων και εγγραφή/αποεγγραφή σε μαθήματα (view_courses.php):**
Αρχικά, αν δεν υπάρχουν διαθέσιμα μαθήματα, εμφανίζεται το αντίστοιχο ενημερωτικό μήνυμα. Αν υπάρχουν, ο φοιτητής μπορεί να επιλέξει μέσω κουμπιού την εγγραφή ή την αποεγγραφή του από κάποιο μάθημα.


* **Προβολή εργασιών (view_assignments.php):**
Στη συγκεκριμένη σελίδα, ο φοιτητής βλέπει τις εργασίες που πρέπει να υποβληθούν, μόνο εφόσον ο καθηγητής έχει αναρτήσει κάποια εργασία. Διαφορετικά, εμφανίζεται το αντίστοιχο μήνυμα. Ο φοιτητής μπορεί να επιλέξει το κουμπί υποβολής, το οποίο τον μεταφέρει στη σελίδα submit_assignments.php.


* **Υποβολή εργασιών (submit_assignments.php):**
Ο φοιτητής μπορεί να επιλέξει το αρχείο προς υποβολή και, πατώντας το κουμπί υποβολής, η εργασία καταχωρείται επιτυχώς στο σύστημα, αλλίως εμφανίζεται αντίστοιχο μήνυμα.



* **Προβολή βαθμολογιών (view_grades.php):**
Ο φοιτητής μπορεί να δει τις βαθμολογίες του, εφόσον είναι διαθέσιμες. Αν δεν υπάρχουν βαθμολογίες, εμφανίζεται σχετικό μήνυμα. Βαθμολογία μεγαλύτερη από 50 εμφανίζεται με πράσινο χρώμα, ενώ βαθμολογία μικρότερη από 50 εμφανίζεται με κόκκινο.


### Καθηγητής

* **Δημιουργία και διαχείριση μαθημάτων (create_manage_courses.php):**
Ο καθηγητής μπορεί να δημιουργεί μαθήματα και να βλέπει τη λίστα των μαθημάτων που έχει δημιουργήσει. Σε περίπτωση που επιχειρήσει να δημιουργήσει μάθημα με όνομα που υπάρχει ήδη, εμφανίζεται κατάλληλο μήνυμα σφάλματος.


* **Ανάρτηση εργασιών (post_assignments.php):**
Ο καθηγητής μπορεί να αναρτήσει εργασία για τα μαθήματα που έχει δημιουργήσει, εισάγοντας τίτλο, περιγραφή και ημερομηνία παράδοσης. Η επιλογή ημερομηνίας γίνεται μέσω Flatpickr (μέσω js/main.js), το οποίο δεν επιτρέπει την επιλογή παρελθοντικών ημερομηνιών. Με την επιλογή του κουμπιού ανάρτησης, η εργασία καταχωρείται επιτυχώς.


* **Προβολή υποβολών φοιτητών (view_submissions.php):**
Ο καθηγητής μπορεί να δει τις υποβολές των φοιτητών, εφόσον υπάρχουν. Σε αντίθετη περίπτωση, εμφανίζεται σχετικό μήνυμα.


* **Βαθμολόγηση φοιτητών ανά μάθημα (grade_students.php):**
Ο καθηγητής μπορεί να βαθμολογεί τους φοιτητές σε κλίμακα από 0 έως 100 και να αποθηκεύει τη βαθμολογία μέσω του αντίστοιχου κουμπιού.

Η πρόσβαση στις λειτουργίες υλοποιείται με **RBAC (Role-Based Access Control)** σε επίπεδο server.

## Δοκιμές (Testing)

Έχουν πραγματοποιηθεί δοκιμές για:

**Εγγραφή χρηστών**
1. Σενάριο: Επιτυχής εγγραφή με σωστό κωδικό → επιτυχής αποθήκευση δεδομένων.
2. Σενάριο: Αποτυχημένη εγγραφή με λανθασμένο κωδικό → εμφάνιση μηνύματος σφάλμα.  

**Σύνδεση / αποσύνδεση**
1. Σενάριο: Σωστή σύνδεση → δημιουργία session και μετάβαση στο dashboard.
2. Σενάριο: Λανθασμένα στοιχεία → παραμονή στη σελίδα login.
3. Σενάριο: Αποσύνδεση → καταστροφή session και ο χρήστης επιστρέφει στη σελίδα σύνδεσης.

**Έλεγχο RBAC**
Πρόσβασης σε μη επιτρεπόμενη σελίδα → “Forbidden Action”. (Μέτα το login)
1. Για-Καθηγητή (μέσω: http://localhost/prototype/php/student/view_courses.php)
2. Για-Φοιτητή (μέσω: http://localhost/prototype/php/teacher/create_manage_courses.php)

**CRUD λειτουργίες Καθηγητή**
1. Σενάριο: Δημιουργία μαθήματος→ Το μάθημα αποθηκεύεται στη βάση.
2. Σενάριο: Δημιουργία μαθήματος με ίδιο όνομα → απορρίψη/εμφανίσει μήνυματος.
3. Σενάριο: Ανάρτηση εργασίας → Εμφανίζεται στους εγγεγραμμένους φοιτητές. 
4. Σενάριο: Προβολή/λήψη υποβολών φοιτητών → Τα αρχεία κατεβαίνουν επιτυχώς.
5. Σενάριο: Βαθμολόγηση εργασίας → Ο βαθμός αποθηκεύεται/εμφανιζεται στον φοιτητή.

**CRUD λειτουργίες Φοιτητή**
1. Σενάριο: Εγγραφή σε μάθημα → Ο φοιτητής εγγράφεται/αποεγγραφεί επιτυχώς.
2. Σενάριο: Προβολή διαθέσιμων εργασιών → Εμφανίζονται εργασίες.
3. Σενάριο: Υποβολή εργασίας → Η υποβολή ολοκληρώνεται/αποθηκεύεται στη βάση.
4. Σενάριο: Επανυποβολή εργασίας → Η προηγούμενη υποβολή αντικαθίσταται.
5. Σενάριο: Προβολή βαθμολογιών → Εμφανίζονται μετά τη βαθμολόγηση.


Η εφαρμογή είναι πλήρως **responsive**. Τα αντίστοιχα screenshots για δοκιμές άλλα και για επίδειξη responsiveness περιλαμβάνονται στο Project Report.
